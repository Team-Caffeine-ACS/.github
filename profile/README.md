## 🏢 Project Overview

The **ACS (Access Control System)** is a digital solution for managing visitors, access rights, and entry/exit events within a secure facility.

### 🔐 Problem Domain

A *pääsla* (gatehouse/reception) is a physical checkpoint where:

- visitors arrive and are registered  
- access rights are granted or restricted  
- hosts or escorts are assigned  
- RFID access cards are issued and returned  

Organizations often rely on manual processes or outdated systems, making it difficult to:

- track who is inside the building  
- manage temporary access cards  
- maintain accurate visit logs  
- ensure security compliance  

### 🎯 Our Goal

To build a **modern, reliable, and user‑friendly visitor management system** that:

- registers arriving and departing visitors  
- logs timestamps for all events  
- tracks which RFID card was issued and when it was returned  
- records who the visitor is meeting or their purpose  
- supports scheduling of future visits  
- improves security through accurate, real‑time data  

---

## 🧩 System Architecture

The ACS system consists of two main repositories:

### **📦 acs-backend**
Backend service responsible for:
- visitor registration  
- access card tracking  
- host assignment  
- visit scheduling  
- data persistence and API endpoints  

**Tech stack:** Node.js, Express, PostgreSQL

---

### **🖥️ acs-frontend**
Frontend application providing:
- visitor check‑in/check‑out interface  
- dashboard for gatehouse personnel  
- forms for scheduling visits  
- real‑time status overview  

**Tech stack:** Next.js, TypeScript, TailwindCSS
